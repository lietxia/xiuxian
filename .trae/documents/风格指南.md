# 修仙项目风格指南

## 1. 文档结构

### 1.1 标题层级

使用以下标题层级结构：

- `#` 一级标题：用于文档主标题
- `##` 二级标题：用于主要章节
- `###` 三级标题：用于章节下的子节
- `####` 四级标题：用于子节下的小节

### 1.2 章节顺序

所有文档应遵循以下基本章节顺序：

1. 目的陈述
2. 定义和特点
3. 分类体系
4. 技术规范
5. 实现说明
6. 使用示例
7. 最佳实践
8. 常见问题
9. 相关资源
10. 更新日志
11. 术语表
12. 联系方式

## 2. 文本格式

### 2.1 字体

- 使用默认字体，避免使用特殊字体
- 正文使用常规字体
- 强调文本使用 `*斜体*` 或 `**粗体**`
- 代码和术语使用 `` `代码/术语` ``

### 2.2 段落

- 段落之间空一行
- 每行长度不超过80个字符
- 避免过长段落，适当分段

### 2.3 换行

- 使用硬换行（每行结束时换行）
- 确保文本在各种编辑器中都能正确显示

## 3. 列表

### 3.1 无序列表

使用 `-` 作为无序列表标记：

```markdown
- 列表项1
- 列表项2
  - 子列表项1
  - 子列表项2
```

### 3.2 有序列表

使用数字和 `.` 作为有序列表标记：

```markdown
1. 第一项
2. 第二项
   1. 子项1
   2. 子项2
```

### 3.3 任务列表

使用 `- [ ]` 和 `- [x]` 作为任务列表标记：

```markdown
- [x] 已完成任务
- [ ] 未完成任务
```

## 4. 表格

### 4.1 基本格式

使用 Markdown 表格格式：

```markdown
| 列1标题 | 列2标题 | 列3标题 |
|---------|---------|---------|
| 内容1   | 内容2   | 内容3   |
| 内容4   | 内容5   | 内容6   |
```

### 4.2 对齐方式

- 左对齐：`| 左对齐 |`
- 居中对齐：`| :居中: |`
- 右对齐：`| 右对齐: |`

### 4.3 表格内容

- 表格内容应简洁明了
- 避免过长内容，必要时使用换行
- 确保表格在各种设备上都能正确显示

## 5. 代码块

### 5.1 基本格式

使用三个反引号 `` ``` `` 作为代码块标记：

```markdown
```
代码内容
```
```

### 5.2 语法高亮

为代码块添加语言标识以启用语法高亮：

```markdown
```javascript
console.log("Hello, world!");
```
```

### 5.3 行号

必要时添加行号：

```markdown
```javascript
1. console.log("Hello, world!");
2. function greet(name) {
3.   return `Hello, ${name}!`;
4. }
```
```

## 6. 引用

### 6.1 块引用

使用 `>` 作为块引用标记：

```markdown
> 这是一个块引用
> 可以包含多行
```

### 6.2 行内引用

使用 `[引用文本](引用来源)` 作为行内引用：

```markdown
这是一个 [行内引用](https://example.com)。
```

## 7. 链接

### 7.1 内部链接

使用相对路径链接到项目内其他文档：

```markdown
[修炼体系](修炼体系/)
[修真境界](修炼体系/境界划分/修真境界.md)
```

### 7.2 外部链接

使用完整 URL 链接到外部资源：

```markdown
[示例网站](https://example.com)
```

### 7.3 锚点链接

使用 `#` 链接到文档内的锚点：

```markdown
[章节1](#1-章节标题)
```

## 8. 图片

### 8.1 基本格式

使用 `![替代文本](图片路径)` 插入图片：

```markdown
![示例图片](images/example.png)
```

### 8.2 图片描述

为所有图片添加适当的替代文本：

```markdown
![修炼境界示意图](images/cultivation-levels.png)
```

### 8.3 图片位置

- 图片应放置在相关文本附近
- 图片下方添加说明文字

## 9. 术语使用

### 9.1 术语一致性

- 所有术语应按照《项目术语表》中的定义使用
- 首次使用术语时应给出完整定义
- 避免使用同义词或近义词

### 9.2 缩写词

- 首次使用缩写词时应给出完整拼写
- 常用缩写词可以直接使用

## 10. 写作风格

### 10.1 语气

- 使用客观、专业的语气
- 避免使用主观、情绪化的语言
- 保持一致的语气

### 10.2 清晰度

- 保持内容清晰明了
- 使用简单、直接的语言
- 避免使用复杂句子结构

### 10.3 准确性

- 确保所有信息准确无误
- 避免模糊或不确定的表述
- 提供具体的数据和例子

### 10.4 一致性

- 保持术语、格式和风格的一致性
- 遵循文档模板
- 使用统一的分类体系

## 11. 文档更新

### 11.1 更新频率

- 定期更新文档，确保信息的时效性
- 重大变更后及时更新文档

### 11.2 更新日志

- 所有文档应包含更新日志
- 记录每次更新的日期、版本、内容和作者
- 更新日志应使用表格格式

## 12. 质量标准

### 12.1 完整性

- 确保文档内容完整，包含所有必要信息
- 避免遗漏重要内容

### 12.2 准确性

- 验证所有信息的准确性
- 检查链接和引用的准确性

### 12.3 一致性

- 确保文档格式和风格一致
- 检查术语使用的一致性

### 12.4 可读性

- 确保文档易于阅读和理解
- 使用适当的标题、列表和表格
- 避免过长段落和复杂句子

## 13. 特殊要求

### 13.1 中文排版

- 中文与英文之间空半角空格
- 中文与数字之间空半角空格
- 避免中英文混排时的格式问题

### 13.2 数字格式

- 使用阿拉伯数字，避免使用中文数字
- 大数字使用千分位分隔符

### 13.3 日期格式

- 使用 `YYYY-MM-DD` 格式表示日期
- 例如：2023-12-31

## 15. 常见错误和避免方法

### 15.1 格式错误

- 检查标题层级是否正确
- 检查列表标记是否一致
- 检查表格格式是否正确

### 15.2 内容错误

- 验证所有信息的准确性
- 检查术语使用是否正确
- 检查链接是否有效

### 15.3 风格错误

- 保持语气一致
- 避免使用主观语言
- 确保文档结构完整
